# Plan D: Simulation and Evaluation Matrix ‚ÄĒ Tier A Comparator

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans.

**Goal:** Implement the Tier A scenario comparator that evaluates every catalog entry against the 9-cell context matrix (3 scales √ó 3 lenses), writing `win | marginal | fail` verdicts + primary numeric metric back into each entry's `results:` block.

**Architecture:** Python package `tier_a` under `simulations/tier-a-comparator/`. Pure-function lens modules (`lenses.py` ‚ÄĒ market, coop, civic). Thresholds from SCALES.md loaded via `scales.py`. Catalog YAML frontmatter read/written by `io_catalog.py`. Runner orchestrates the 15 √ó 9 matrix via `runner.py`; CLI entry via `cli.py`. TDD: failing tests land first, impl makes them pass.

**Tech Stack:** Python 3.12+, PyYAML 6.x, pytest 9.x, stdlib only otherwise. No deps beyond pytest + PyYAML.

**Source of truth:**
- `corpus/program/ECONOMIC-LENSES.md` ‚ÄĒ lens rules (authoritative)
- `corpus/program/SCALES.md` ‚ÄĒ threshold values per scale
- `catalog/SCHEMA.md` + `catalog/smithing/SCHEMA.md` ‚ÄĒ frontmatter schema
- `catalog/smithing/entries/*.md` ‚ÄĒ 15 entries as input

**Style:** Python PEP-8. Type hints on all public functions. Docstrings with cited ECONOMIC-LENSES rule for each formula. Tests first.

**Commit cadence:** One commit per logical unit (scaffold ‚Üí red tests ‚Üí green impl ‚Üí runner ‚Üí writeback).

## Lens Pass Conditions (from ECONOMIC-LENSES.md)

**MARKET:** payback ‚Č§ 8 years AND operator_take_home ‚Č• scale_median_wage
- Inputs: `capital_cost.mid`, `annual_costs` (maintenance + consumables + rent), `throughput_units_equiv_per_year`, `market_price_per_unit.mid`, `labor_hours_per_unit`, `wage_rate_from_scales`, `downtime_fraction`
- Primary metric: payback years

**COOP:** break_even_member_count ‚Č§ feasible_member_pool(scale)
- Inputs: `capital_cost.mid`, annual_costs, member_dues (assumed from SCALES participation rates), scale population
- Primary metric: break-even member count

**CIVIC:** per_household_cost ‚Č§ scale_threshold AND utilization ‚Č• scale_utilization_threshold
- Inputs: `capital_cost.mid` amortized over `lifespan_years` (25-40 bond), annual_costs, scale_household_count, public_use_hours (inferred from max_active_hours)
- Primary metric: per-household annual cost

**Verdict states:** `win` = pass with ‚Č•20% margin. `marginal` = pass with <20% margin OR optimistic assumption. `fail` = does not pass.

## Retrospective: Risks (upfront)

- **Lens-formula correctness.** The pass conditions are stated in ECONOMIC-LENSES.md but the exact arithmetic (wage derivation, capital amortization, participation-rate math) has degrees of freedom. Tests pin the interpretation. If a reviewer disagrees with a formula choice, the fix is to update the test + implementation together.
- **Catalog-entry data quality.** 450+ `[CITATION-NEEDED]` placeholders across 15 entries means sim output is only as good as inferred inputs. The SUMMARY should note this explicitly; verdicts are provisional pending the verification queue from Plan C.
- **SCALES thresholds are illustrative.** Per SCALES.md, the wage/civic/participation values are labeled "illustrative anchors" ‚ÄĒ downstream consumers of Tier A output must carry this caveat.
- **Edge cases in YAML roundtrip.** PyYAML's default loader can mangle ordering, quote styles, booleans. Use `yaml.safe_load` + `yaml.safe_dump` with `sort_keys=False`. If roundtrip drift is significant, consider `ruamel.yaml` later (deferred).

## Failure Modes

- **If lens formulas produce clearly wrong verdicts on sanity-check entries** ‚ÄĒ flag and halt; formulas need revision before the full run.
- **If YAML roundtrip mangles catalog files** ‚ÄĒ back out the writeback; investigate; consider using a targeted `results:` block edit rather than full-file rewrite.
- **If a catalog entry has invalid/missing sim_params** ‚ÄĒ skip with a logged warning; don't crash; collect skipped-entry count into SUMMARY.
