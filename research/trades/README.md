# research/trades/

Per-trade functional-requirements packages. Each subdirectory contains three
files: REQUIREMENTS.md (what the equipment must do), HISTORICAL-FORMS.md
(documented historical variants), and SOURCES.md (citations).

Convention: one subdirectory per trade, lowercase hyphenated name (e.g.,
`smithing/`, `baking/`, `weaving/`). Later trades are stubbed here and
filled when their plan runs.

Populated by Plan B — functional requirements for smithing first.
